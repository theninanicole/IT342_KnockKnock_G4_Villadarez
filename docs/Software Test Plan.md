# Knock Knock Software Test Plan

## 1. Project Information

| Field | Details |
| --- | --- |
| Project Name | Knock Knock Visitor Management System |
| System Type | Web, Backend API, and Android Mobile Application |
| Backend | Spring Boot |
| Web Frontend | React + Vite |
| Mobile | Android Kotlin |
| Test Plan Version | 1.0 |
| Date | May 9, 2026 |

## 2. Purpose

This Software Test Plan defines the testing procedures for the implemented Knock Knock system. It covers the currently available functional requirements, test cases, manual test steps, and automated test cases for the major modules.

## 3. Scope of Testing

The testing scope is limited to features currently implemented in the project.

### Included Features

- Visitor registration and login.
- Condominium administrator registration and login.
- Google authentication API support.
- JWT-based protected API access.
- Visitor visit scheduling.
- Visitor visit viewing, editing, and cancellation.
- QR code generation and QR email sending.
- Admin visit viewing and status filtering.
- Admin visit verification by reference number or QR scanner.
- Admin visitor check-in and check-out.
- Visit status history.
- Notifications and mark-as-read.
- User profile update and password change.
- Condominium list retrieval.
- Android visitor login, registration, session storage, dashboard visit loading, and notification loading.

### Not Included

- Performance/load testing.
- Production deployment testing.
- Full live Google, email, or Supabase availability testing beyond configured integration checks.

## 12. Completion Criteria

Testing is complete when:

- All implemented functional requirements have at least one test case.
- Manual test steps have been executed and results recorded.
- Automated checks for backend, web, and mobile have been run where available.
- Failed or blocked tests are documented with issue details.
- Critical features such as login, visit creation, admin check-in/check-out, and protected access are verified.
